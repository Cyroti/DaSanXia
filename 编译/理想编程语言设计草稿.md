# 理想编程语言设计：Cyroti

**23302010049 黄文峰**

## 1. 语言名称

我给这门语言起名叫 **Cyroti**， 名字本身没有什么特殊的缩写或者递归定义。

## 2. 设计理念

Cyroti 主要想解决的问题，是很多语言在“写起来轻松”和“读起来清楚”之间很难兼顾：有的语法太重，有的又过于自由，最后让代码越来越乱。它和现有语言相比，整体外观会更整齐清楚，同时保留比较轻量、直接的表达方式。 总体而言，他是类C的，但也吸收了一些更加现代更加清楚的表达方式。

## 3. 基本语法定义

Cyroti 使用花括号表示代码块，句末保留分号，整体风格偏清晰、直接。

### 3.1 变量声明

Cyroti 里我把变量分成两种：一种是定义之后不再改的，一种是后面还会变的。

- `let`：不可变变量
- `slot`：可变变量

```cyroti
let name = "Cyroti";
let year: Int = 2026;
slot score = 0;
slot price: Float = 19.9;
```

这样设计是为了让“这个值后面会不会变”一眼就能看出来。

### 3.2 基本类型

Cyroti 先提供一些最基本、最常用的类型：

- `Int`：整数
- `Float`：浮点数
- `Bool`：布尔值
- `Text`：字符串
- `List<T>`：列表
- `Map<K, V>`：映射
- `None`：空值

示例：

```cyroti
let age: Int = 20;
let pi: Float = 3.14;
let passed: Bool = true;
let title: Text = "hello";
let nums: List<Int> = [1, 2, 3];
let user: Map<Text, Int> = {"level": 3, "coin": 99};
```

如果编译器能推断出类型，也可以省略类型标注。

### 3.3 赋值语句

赋值本身比较普通，和多数语言差不多：

```cyroti
slot total = 10;
total = total + 5;
```

但是 `let` 声明的变量不能重新赋值：

```cyroti
let max = 100;
// max = 120;   非法
```

这样可以减少变量被随意改动带来的错误。

### 3.4 控制流语句

#### 条件语句

普通的条件分支写法如下：

```cyroti
if (score >= 60) {
	print("pass");
} else {
	print("fail");
}
```

Cyroti 里 `if` 不只是控制流程，也可以直接产生一个结果。

```cyroti
let grade = if (score >= 90) {
	"A"
} else if (score >= 60) {
	"B"
} else {
	"C"
};
```

这比先定义变量、再在不同分支里赋值更紧凑。

#### 循环语句

Cyroti 支持 `while`，也支持更自然一点的 `for in`：

```cyroti
slot i = 0;
while (i < 3) {
	print(i);
	i = i + 1;
}

for n in [1, 2, 3, 4] {
	print(n);
}
```

另外也支持区间：

```cyroti
for i in 1..5 {
	print(i);
}
```

循环部分我希望尽量简单直接，不加入太复杂的特殊形式。

### 3.5 函数声明与调用

Cyroti 用 `fn` 定义函数；参数和返回值的标注方式则保持得比较直观。

```cyroti
fn add(a: Int, b: Int) -> Int {
	a + b
}

let result = add(3, 4);
print(result);
```

这里涉及到 Cyroti 的一个重要规则：**隐式返回**。

1. **通常不需要 `return`**：函数体会自动返回**最后一行表达式的值**。只要把它放在最后，Cyroti 就会知道这是你要返回的结果。
2. **`return` 仅用于提前退出**：`return` 关键字依然存在，但只有当你需要在代码中间（比如满足某个条件就立刻结束）提前跳出函数时，才需要用到它。

**如何区分“代码块的值”和“函数返回值”？**

- 代码块（如 `if`）产生的值，只是这个块本身的值，它可以参与运算或赋给变量。
- 只有当这个值处于函数体的**最后一行**时，它才会被当作函数的返回值。

示例说明：

```cyroti
fn complex_logic(x: Int) -> Int {
    // 这里的 if 产生值 100 或 0，赋给了变量 t，函数还没结束
    let t = if (x > 10) { 100 } else { 0 };
    
    if (x < 0) {
        return 0; // 情况特别，需要由 return 提前终止函数
    }

    t + x // 这是最后一行，它的计算结果会自动返回
}
```

### 3.6 可选内容：面向对象编程

Cyroti 也保留了轻量的类，用来处理需要封装状态的情况。

```cyroti
class Counter {
	slot value: Int = 0;

	fn inc() -> Int {
		value = value + 1;
		value
	}
}

let c = Counter();
print(c.inc());
```

类不是必须的核心，但在需要封装状态时会比较方便。

## 4. 与常见编程语言不同的设计特点,也可以说是一些新奇的写法

### 特点一：代码块默认可以产生值

比如在 Java 或 JavaScript 里，`if` 更多只是控制流程；但在 Cyroti 里，它本身也能当成表达式使用。

```cyroti
let tag = if (age >= 18) {
	"adult"
} else {
	"minor"
};
```

这样可以省掉很多临时变量，写判断时也更集中。

### 特点二：内置管道传值语法

Cyroti 支持 `|>` 管道运算符，前一步的结果会自动流到后一步里。

```cyroti
let result = [1, 2, 3, 4]
	|> map(fn(x: Int) -> Int { x * 2 })
	|> filter(fn(x: Int) -> Bool { x > 4 })
	|> sum();
```

这样写的好处是数据处理过程能从上到下顺着看，比一层层函数嵌套更直观。

### 特点三：加入 `need` 前置约束语句

Cyroti 里还加入了一个 `need`，专门用来写前置条件。

```cyroti
fn divide(a: Int, b: Int) -> Int {
	need b != 0 : "b cannot be zero";
	a / b
}
```

它和普通 `if` 不太一样，重点是把“前置条件”单独写出来，让函数入口更清楚。

## 5. 示例程序

下面是一段用 Cyroti 写的阶乘程序：

```cyroti
fn fact(n: Int) -> Int {
	if (n <= 1) { 1 } else { n * fact(n - 1) } //这里用到了代码块可以产生值的特性，产生的值隐式返回了
}

let input = 5;
let answer = fact(input);
print(answer);
```

这段代码里能看到变量声明、函数定义、条件表达式和递归调用。

## 6. 总结

总的来说，Cyroti 是一门强调“好写、好读、不过分复杂”的语言。它希望在清晰性和灵活性之间找到一个更自然的平衡，让代码既有表达力，又不会太难理解。
